# Тест-кейсы для мануального тестирования

Все тест-кейсы выполняются в тестовом Telegram-чате с ботом в `TEST_MODE=true`.

---

## TC-01 — Добавление бота в чат

**Предусловие:** бот не в чате  
**Действие:** добавить бота в групповой чат

**Ожидаемый результат:**
- [ ] Бот отправляет сообщение с кнопками `[Приєднатися до гри]` `[Техпідтримка]` `[Вийти з гри]`
- [ ] Через 1 секунду появляется второе сообщение с кнопкой `[Розпочати гру]`
- [ ] В БД создана запись в `games` со статусом `pending`
- [ ] Юзер, добавивший бота, записан в `players` как админ игры <!--  обновил: юезр добавляется как админ, а не как обычный юзер-->
- [ ] В логах: `[INFO] [chat:ID] game created, admin: user_id:username, game: id:chatname` <!-- добавил в логи данные игры, айди и имя чата-->

## TC-20 — Сабтаска 2: эксклюзивный лок

**Предусловие:** опубликована таска с subtask типа `voting_collage`

1. Юзер A нажимает `[Хочу відповісти]` → лок захвачен, `task_locks` создан
2. Юзер B нажимает `[Хочу відповісти]` пока A отвечает

**Ожидаемый результат для шага 2:**
- [ ] Бот тегает юзера B, вариативное сообщение о том, что другая подруга сейчас отвечает на эту таску
- [ ] Через 10 секунд это сообщение удаляется
- [ ] Лок НЕ передан юзеру B
- [ ] `task_locks` — запись по-прежнему на юзера A

3. Юзер A отвечает на все вопросы сабтаски
- [ ] Лок освобождается (`task_locks` удаляется)
- [ ] Бот тегает юзера A, благодарит за ответ и сообщает, что когда все ответят — будет коллаж
